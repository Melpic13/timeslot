# Contributing

## Development setup

1. Install Go 1.22+
2. Clone the repository
3. Run `go test ./...`

## Code style

- Follow standard Go conventions
- Keep API docs on exported symbols
- Prefer table-driven tests for behavior coverage

## Testing requirements

- Run `go test ./...` before opening a PR
- Add tests for any new feature/bug fix

## PR process

1. Open an issue when needed
2. Submit a focused PR with tests
3. Ensure CI passes

## Issue guidelines

Use bug/feature templates under `.github/ISSUE_TEMPLATE`.
